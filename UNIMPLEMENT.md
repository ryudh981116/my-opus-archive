## 📝 상세 설명

### 1️⃣ 연주 내역 수정 (EDIT)
**우선순위**: 🔴 High | **난이도**: ★★☆☆☆

#### 목표
사용자가 이미 저장된 연주 내역을 모두 수정할 수 있는 기능

#### 상세 요구사항
- "내 연주 내역" 페이지에서 ✏️ 수정 버튼 클릭
- 새 연주 기록과 동일한 폼 표시 (기존 데이터 미리 채워짐)
- 수정 후 저장 → 데이터 업데이트
- 최종 수정 시간 기록 (`updated_at`)

#### 기대 효과
- 사용자 경험 개선
- 데이터 신뢰도 향상

#### 예상 개발 시간
- 약 2-3시간

---

### 2️⃣ 비밀번호 해싱 (Security Enhancement)
**우선순위**: 🔴 High | **난이도**: ★★★☆☆

#### 목표
평문 비밀번호를 암호화하여 보안 강화

#### 상세 요구사항
- `bcrypt` 라이브러리 사용
- 회원가입: 비밀번호 해싱 → 저장
- 로그인: 입력 비밀번호 해싱 → 저장된 해시와 비교
- 기존 사용자 데이터 마이그레이션 (선택사항)

#### 구현 예시
```python
import bcrypt

# 회원가입
hashed = bcrypt.hashpw(password.encode(), bcrypt.gensalt())

# 로그인
if bcrypt.checkpw(input_password.encode(), stored_hash):
    # 로그인 성공
```

#### 기대 효과
- 사용자 정보 보호
- 규정 준수 (보안 기준)

#### requirements.txt 추가
```
bcrypt==4.1.2
```

### 3️⃣ 연주 내역 내보내기 (Export)
**우선순위**: 🟠 Medium | **난이도**: ★★★☆☆

#### 목표
연주 내역을 CSV/PDF 형식으로 다운로드

#### 상세 요구사항
- "내 연주 내역" 페이지 상단에 "📥 내보내기" 드롭다운
- 형식 선택: CSV / PDF
- CSV: pandas 사용
- PDF: reportlab 또는 fpdf2 사용

#### 데이터 구성
```
날짜, 장소, 악기, 파트, 단체명, 지휘자, 공개여부, 페이, 곡목
2025-02-13, 예술의전당, 바이올린, 1st Violin, 서울 필, 라포, 공개, 0, 1부: 협주곡 | 2부: 소나타
```

#### 기대 효과
- 사용자 만족도 향상
- 백업/인쇄 용도

#### requirements.txt 추가
```
pandas==2.1.4
fpdf2==2.7.0
```

### 4️⃣ 데이터베이스 마이그레이션
**우선순위**: 🔴 High | **난이도**: ★★★★☆

#### 목표
JSON 기반 저장소 → 관계형 DB (PostgreSQL/SQLite)로 전환

#### 상세 요구사항
- SQLAlchemy ORM 사용
- 테이블 설계:
  - `users` (id, username, email, password_hash, created_at)
  - `performances` (id, user_id, date, venue, instrument, ...)
  - `comments` (id, performance_id, user_id, content, created_at)
  - `likes` (id, performance_id, user_id, created_at)
- 마이그레이션 스크립트 제공
- 기존 JSON 데이터 자동 변환

#### 기대 효과
- 데이터 무결성 보장
- 쿼리 성능 향상
- 확장성 증가

#### requirements.txt 추가
```
sqlalchemy==2.0.23
psycopg2-binary==2.9.9  # PostgreSQL
```

### 5️⃣ 사용자 프로필 페이지
**우선순위**: 🟠 Medium | **난이도**: ★★☆☆☆

#### 목표
사용자별 프로필 정보 및 통계 표시

#### 상세 요구사항
- 새 메뉴 추가: "프로필" 또는 "마이페이지"
- 프로필 정보:
  - 사용자명 (👤)
  - 가입 날짜 (📅)
  - 총 연주 내역 수 (📊)
  - 공개된 연주 수 (🌍)
  - 받은 좋아요 수 (❤️)
  - 받은 댓글 수 (💬)
  - 자기소개 (bio) - 선택사항
- 프로필 수정 버튼

#### 통계 대시보드 예시
```
🎼 My Opus Archive Stats
─────────────────────────
📝 총 연주 내역: 25건
🌍 공개 연주: 18건
❤️ 받은 좋아요: 142개
💬 받은 댓글: 37개
📅 가입일: 2025-02-13
```

#### 기대 효과
- 사용자 만족도 향상
- 커뮤니티 참여도 증진

- "➕ 팔로우" 버튼 추가

      "created_at": "2025-02-13T10:30:45"

  }

#### 기대 효과
- 커뮤니티 네트워킹

### 7️⃣ 실시간 알림 (Notification)
**우선순위**: 🟠 Medium | **난이도**: ★★★☆☆

#### 목표
사용자의 활동(댓글, 좋아요 등)에 대한 실시간 알림 제공

#### 상세 요구사항
- 알림 유형:
  - "팔로우 중인 사용자가 새 연주를 올렸습니다"
  - "당신의 연주에 댓글이 달렸습니다: {content}"
  - "당신의 연주에 좋아요를 받았습니다"
- 알림 센터 (사이드바 상단)
- 읽음/안 읽음 표시
- 알림 삭제

#### 데이터 구조
```json
{
  "notification_123": {
    "recipient_id": "username",
    "type": "comment|like|follow",
    "data": {...},
    "is_read": false,
    "created_at": "2025-02-13T10:30:45"
  }
}
```

#### 기대 효과
- 사용자 재방문율 증가 (50%↑ 예상)
- 플랫폼 활동성 향상
